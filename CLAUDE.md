## Architecture

### Data flow

```
assets/*.json  →  MigrationService  →  IsarService  →  DataService  →  UI pages
```

On first launch, `MigrationService` reads the JSON asset files and populates the Isar database. Subsequent launches skip migration. `DataService` is the single entry point for all UI data access — UI pages never call `IsarService` directly.

### Layer structure

**`lib/data/`**
- `models/` — Isar `@collection` classes: `Muscle`, `Exercise`, `Program`/`Session`, `SessionLog`. Each model has a generated `.g.dart` file (do not edit manually).
- `services/data_service.dart` — Static service exposing `ProgramData`, `SessionData`, `ExerciseData` DTOs to the UI. Also exposes workout logging (`logWorkoutSession`) and history retrieval.
- `services/isar_service.dart` — Thin wrapper around the Isar instance; handles all DB reads/writes.
- `services/migration_service.dart` — One-time JSON-to-Isar migration on first install.
- `models_isar_backup/` and `services_isar_backup/` — Old backup copies; ignore unless reverting.

**`lib/ui/`**
- Pages: `ProgramListPage` → `SessionListPage` → `SessionOverviewPage` → `WorkoutPage`
- `theme/` — Design system tokens: `AppColors` (black/white strict palette), `AppTypography`, `AppDimensions` (breakpoints + responsive helpers), `AppSpacing`, `AppShadows`, `AppAnimations`, `AppTextStyles`, `AppTheme`
- `widgets/` — Reusable components: `AppButton`, `AppCard`, `AppChip`, `AppScaffold`, `RepCounter`, `RepControls`, `RestTimer`, `ExerciseProgressHeader`, `ResponsiveBuilder`
- `animations/` — `BreathingAnimation`, `EffortAnimation`, `FadeIn`, `SlideTransition`, `AppTransitions`

**`lib/utils/`**
- `TimerNotifier` — `ValueNotifier<TimerState>` driving both exercise countdowns and rest timers. `WorkoutPage` listens to it via `ValueListenableBuilder`.
- `AppExceptions` — Typed exception hierarchy (`DataLoadingException`, `NotFoundException`, `DataParsingException`).
- `ErrorHandler` — Centralized error display (SnackBar + error widgets).

**`lib/l10n/`** — FR/EN localizations generated by Flutter's `intl` tool. Add strings to `.arb` files, then run `flutter gen-l10n`.

### Navigation (go_router)

```
/                 → ProgramListPage
/sessions         → SessionListPage
/session/:id      → SessionOverviewPage
/workout/:sessionId → WorkoutPage
```

### Design system rules

- **Black & white only** — `AppColors.black` / `AppColors.white` for all structural UI. Semantic colors (`success`, `error`, etc.) exist in `AppColors` but are reserved for future use.
- All spacing via `AppSpacing` constants; all dimensions via `AppDimensions`. Use `AppDimensions.*Responsive(screenWidth)` helpers for adaptive layouts via `ResponsiveBuilder`.
- Breakpoints: small `< 600`, medium `600–900`, large `> 900`.

### Adding or modifying Isar models

1. Edit the class in `lib/data/models/`.
2. Run `dart run build_runner build --delete-conflicting-outputs` to regenerate `.g.dart`.
3. If the schema changes break existing persisted data, increment the Isar schema version in `IsarService.initialize()` and add a migration.

### Data assets

Training data lives in `assets/` as JSON and is never modified at runtime:
- `muscles.json` — muscle catalog
- `exercises.json` — exercise catalog  
- `programme.json` — legacy format
- `programmes/chaine_porteuse.json` — current structured program
